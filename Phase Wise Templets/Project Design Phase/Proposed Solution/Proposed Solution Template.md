| S.No. | Parameter | Description |
| :--- | :--- | :--- |
| 1 | **Problem Statement (Problem to be solved)** | Customers experience slow shopping response times, dynamic filter lag, and lost cart sessions on legacy e-commerce systems, leading to high bounce and cart abandonment rates. Store admins also lack a user-friendly system to manage products and track customer orders without direct DB modifications. |
| 2 | **Idea / Solution description** | **ShopEZ**: A fast MERN-stack web application designed for seamless customer shopping and administrative control. Key features include a persistent database-backed cart, dynamic frontend filtering, role-based JWT authentication, order placement workflows, and a custom admin portal for catalog and banner management. |
| 3 | **Novelty / Uniqueness** | ShopEZ utilizes a fully asynchronous MERN Single Page Application (SPA) architecture that eliminates page reloads. It implements React's Context API along with automated Axios request interceptors to seamlessly handle roles, authorization levels, and persistent user states. |
| 4 | **Social Impact / Customer Satisfaction** | Provides small to medium retail vendors and consumers a premium, high-performance, and secure interface. This improves accessibility, speeds up daily digital transactions, reduces checkout friction, and guarantees data integrity. |
| 5 | **Business Model (Revenue Model)** | Subscription/licensing model for retail businesses wishing to deploy a digital storefront (Software-as-a-Service) or transactional commissions on sales processed through the platform. |
| 6 | **Scalability of the Solution** | Highly scalable. The decoupled frontend (React/Vite) can be distributed via CDNs. The Express API server can be containerized using Docker and auto-scaled behind load balancers. MongoDB accommodates heavy data write volumes using horizontal sharding. |
